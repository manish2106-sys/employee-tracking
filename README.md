# Employee Tracking System

Full-stack project with:
- Android app for employees (check-in/check-out)
- Web admin dashboard
- Backend REST API

## Live Production Access (from anywhere)

This repo is configured for **Render Blueprint deployment** with:
- Public backend API service
- Public admin dashboard service

Blueprint file:
- `render.yaml`

### One-click Render Deploy

Open this link and deploy:

`https://dashboard.render.com/blueprint/new?repo=https://github.com/manish2106-sys/employee-tracking`

After deploy completes:
- Admin dashboard URL: `https://manish2106-employee-tracking-admin.onrender.com`
- API health URL: `https://manish2106-employee-tracking-api.onrender.com/health`

### Required secrets while creating Blueprint
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`

`JWT_SECRET` is auto-generated by Render.

## Android App Production URL

App default backend URL is now:
- `https://manish2106-employee-tracking-api.onrender.com/api`

So employees can check in/check out from any internet connection.

## Features

### Android Employee App
- Check In / Check Out actions
- Captures:
  - Employee ID
  - GPS coordinates
  - Device name
  - Login/Logout timestamps (stored server-side)

### Web Admin Panel
- Admin login
- Employee CRUD + activate/deactivate
- Attendance sessions table
- Dashboard summary cards
- Date range filtering

### Backend API
- JWT admin auth
- Employee and attendance APIs
- File-based persistence: `backend/data/db.json`

## Local Run

### 1) Backend
PowerShell:
```powershell
cd backend
Copy-Item .env.example .env
npm.cmd install
npm.cmd run dev
```

### 2) Web Admin
```powershell
cd web-admin
node dev-server.mjs
```
Open: `http://localhost:3000`

### 3) Android App
Open `android-app` in Android Studio and run.

## API Endpoints

### Public attendance
- `POST /api/attendance/check-in`
- `POST /api/attendance/check-out`

### Admin
- `POST /api/admin/login`
- `GET /api/admin/summary`
- `GET /api/admin/employees`
- `POST /api/admin/employees`
- `PUT /api/admin/employees/:id`
- `DELETE /api/admin/employees/:id`
- `GET /api/admin/attendance`

## Note

For strict enterprise production, next upgrade is moving persistence from JSON file to managed PostgreSQL.
